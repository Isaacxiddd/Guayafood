### 2. MCP de Mercado Pago — token vencido en producción

El `MERCADOPAGO_ACCESS_TOKEN` en Vercel tenía el token viejo. Se detectó vía logs del deployment con el Vercel MCP.

**Fix:** renovado usando la CLI de Vercel:
```bash
npx vercel env rm MERCADOPAGO_ACCESS_TOKEN production
echo "nuevo_token" | npx vercel env add MERCADOPAGO_ACCESS_TOKEN production
npx vercel redeploy <deployment-id>
```

**Para el futuro:** cuando el token expire, repetir estos 3 comandos. El token activo siempre está en `.env` local.

### 6. Pentesting — seguridad de los endpoints que escriben en Google Sheets

Se escribió `tests/pentest-sheet.mjs` con 10 vectores de ataque y 38 assertions.
Correr con: `node tests/pentest-sheet.mjs` (requiere `pnpm dev` activo).

#### Bugs encontrados y corregidos

**`src/pages/api/save-order.ts` — `sanitizeSheetValue`**

| Bug | Fix |
|---|---|
| `\t=IMPORTRANGE(...)` y `\r=formula` no eran detectados (el regex solo chequeaba el primer char) | `trimStart()` antes del regex de fórmulas |
| Null-bytes `\x00` y chars de control pasaban a Google Sheets | Strip de `[\x00-\x08\x0B\x0C\x0E-\x1F\x7F]` |
| Tipos no-string (números, booleans, objetos) llegaban a `sanitizeSheetValue` sin castear | `String(value ?? '')` como primer paso |

**`src/lib/rate-limit.ts` — `getClientIp`**

| Bug | Fix |
|---|---|
| Usaba el primer valor de `X-Forwarded-For`, que el cliente puede falsificar para rotar el rate limiter | Prioriza `x-real-ip` (seteado por Vercel edge, no spoofeable); usa el **último** valor de XFF como fallback |

#### Warnings informativos (no requieren acción)

- **Origin spoofing server-to-server:** `checkOrigin` protege solo CSRF de browser. Un atacante con curl puede setear `Origin` manualmente — es una limitación conocida del modelo, no un bug.
- **`check-availability` expone `currentCount`:** informa cuántos pedidos hay en un slot, pero no expone datos de clientes. Bajo impacto.
- **Campos sin límite de longitud:** cubiertos por rate limit y validación de origen.
- **Formula injection "llega al servidor":** esperado — se sanitiza antes de escribir en el sheet. El 200 es correcto.
